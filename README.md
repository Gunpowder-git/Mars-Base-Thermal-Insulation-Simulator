# 火星基地多材料隔热墙优化 · 热设计工作室

物理项目化学习 —— 用热传导元胞自动机模拟火星基地隔热墙设计。

## 项目结构

```
├── 热传导.py                   原始基础版（固定场景演示）
├── run_studio.py               tkinter 桌面版启动器
├── thermal_design_studio.py    tkinter 桌面版核心引擎
│
├── app.py                      Flask Web 版后端（当前主力）
├── 启动Web界面.py               Web 版一键启动脚本
├── templates/
│   └── index.html              前端页面
├── static/
│   ├── style.css               暗色火星主题样式
│   └── app.js                  Plotly.js 交互逻辑
│
├── requirements.txt             Python 依赖
└── README.md                   本文件
```

## 三个版本

| 版本 | 启动命令 | 说明 |
|------|----------|------|
| 原始版 | `python 热传导.py` | 最早的演示，固定场景，matplotlib 动画 |
| 桌面版 | `python run_studio.py` | tkinter GUI，材料选择 + 缺陷 + 报告 |
| **Web 版** | `python app.py` 或双击 `启动Web界面.py` | 浏览器运行，专业界面，功能最全 |

## Web 版功能

浏览器打开 `http://localhost:5000`

### 控制面板
- 5 层墙体材料自由组合（8 种材料可选）
- 预设方案一键切换：基础 / 进阶 / 优化
- 环境参数滑块：室温目标、火星温度范围
- 缺陷模拟：热桥（铜导热层）/ 裂缝（空气渗透）
- 启动 / 暂停 / 重置仿真

### 可视化
- 温度场交互式热力图（Plotly.js，可缩放悬停）
- 墙体中心线温度剖面曲线
- 材料色块图例

### 实时物理量
- 室温 · 热阻 R · 热流密度 q · 温差 ΔT
- 材料成本 · 墙体重量 · 温度波动 · 墙体均温

### 增强模式
- **故事模式**：5 个场景自动演示（火星环境 → 无保温 → 基础保温 → 优化设计 → 热桥缺陷）
- **对比模式**：基础方案与优化方案并排对比

## 材料库

| 材料 | 热扩散系数 α | 热导率 k (W/m·K) | 密度 (kg/m³) | 成本 |
|------|-------------|------------------|-------------|------|
| 空气 | 0.15 | 0.026 | 1.2 | 0 |
| 混凝土 | 0.008 | 1.4 | 2400 | 100 |
| 保温泡沫 | 0.001 | 0.035 | 50 | 300 |
| 气凝胶 | 0.0003 | 0.015 | 20 | 500 |
| 陶瓷 | 0.002 | 0.8 | 1200 | 200 |
| 金属绝缘 | 0.0005 | 0.005 | 100 | 400 |
| 真空 | 0.0001 | 0.0001 | 0 | 1000 |
| 铜导热层 | 0.12 | 400 | 8960 | 600 |

## 物理原理

基于二维热传导方程（傅里叶定律）的离散化：

> ∂T/∂t = α · ∇²T

- 采用显式有限差分法，CFL 条件控制时间步长
- 矢量化 numpy 实现（`app.py` 内嵌引擎）
- 边界条件：Dirichlet（左/右边界固定温度）+ Neumann（上/下绝热）
- 火星外部温度采用正弦昼夜循环模型

## 评估指标

- **R 值**（热阻）：R = ΔT / q，衡量墙体保温能力
- **热流密度 q**：q = -k · ∇T，单位面积热量传导速率
- **温度稳定性**：室内温度标准差
- **成本效益**：材料总成本
- **结构重量**：墙体总重量

## 开发背景

从 `热传导.py`（通用热传导演示）逐步演进：
1. 原始版 → 固定场景，验证热传导元胞自动机可行性
2. 桌面版 → 加入材料库、火星环境、交互式设计、报告系统
3. Web 版 → 浏览器界面，Plotly.js 可视化，故事/对比模式，适合演示和录课
